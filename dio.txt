# 📚 Miniguia de Estudos com NotebookLM: Otimização de RAG e Semantic Routers

## 🎯 Contexto e Objetivos
Este repositório documenta a criação de um Caderno Temático focado em **LLM Ops**, especificamente na arquitetura de **Retrieval-Augmented Generation (RAG)** e implementação de **Semantic Routers**. 

**Objetivos de Estudo:**
* Compreender as melhores práticas para fragmentação de texto (chunking) e vetorização em bases de conhecimento.
* Explorar como roteadores semânticos podem direcionar intenções de usuários para pipelines específicos (ex: suporte vs. vendas).
* Utilizar o **NotebookLM** da Google como ferramenta de aprendizagem ativa para cruzar informações de documentações técnicas complexas.

## 🛠️ Engenharia de Prompts e "Cicatrizes" (Troubleshooting)
Nesta etapa, apliquei testes de estresse no NotebookLM para extrair resumos precisos, lidando com jargões técnicos e evitando alucinações.

### Teste 1: Compreensão Básica
* **Prompt:** "Com base nos documentos, explique a principal diferença entre um RAG padrão e um pipeline que utiliza Semantic Routers antes da geração de resposta."
* **Resultado:** A IA deu uma resposta correta, mas muito focada em latência e pouco em precisão de intenção.
* **Cicatriz/Ajuste:** Percebi que o prompt foi muito amplo. Precisava forçar a IA a olhar para o artigo do LangChain. 
* **Prompt Refinado:** "Aja como um Arquiteto de IA. Cruzando o Paper de RAG e a documentação do Semantic Router, crie uma tabela comparativa listando as vantagens de usar roteamento semântico no início de um pipeline de suporte, focando em redução de custos e precisão de contexto." (Sucesso!)

### Teste 2: Geração de Código/Lógica
* **Prompt:** "Me dê um exemplo de código para um router semântico simples."
* **Resultado:** A IA tentou inventar uma biblioteca inexistente baseada em um trecho de texto solto.
* **Cicatriz/Ajuste:** O NotebookLM é focado em leitura dos PDFs, não em codificação direta. Mudei a abordagem para extrair a lógica e não o código.
* **Prompt Refinado:** "Com base no Guia do Semantic Router, liste o passo a passo lógico (em formato de algoritmo ou diagrama de texto) de como uma query de usuário é convertida em embedding e roteada para o pipeline correto."

## 🧠 Miniguia de Estudo (Entrega Final)

### 1. Resumo Estruturado: O Fluxo de um Chatbot Escalável
A integração de RAG com Roteadores Semânticos resolve o problema do "chatbot que tenta fazer tudo ao mesmo tempo". 
* **A Rota:** Quando o usuário envia uma mensagem, um *encoder* rápido transforma o texto em um vetor e o compara com "rotas" pré-definidas (ex: Dúvidas Técnicas, Financeiro, Bate-papo).
* **A Recuperação (RAG):** Se a rota escolhida for "Dúvidas Técnicas", a query é enviada para um banco de dados vetorial específico daquele contexto, que recupera os trechos mais relevantes dos manuais.
* **A Geração:** O LLM recebe a query original + os trechos recuperados e gera uma resposta embasada, minimizando o risco de alucinações.

### 2. Glossário
* **Embedding:** Representação matemática (vetorial) do significado de um texto.
* **Semantic Router:** Camada de decisão rápida que direciona prompts para diferentes fluxos ou modelos com base na proximidade semântica, sem precisar chamar um LLM pesado para classificar a intenção.
* **Chunking:** Processo de dividir documentos grandes em pedaços menores para facilitar a indexação e a busca no banco de dados vetorial.
* **Vector Database:** Banco de dados otimizado para armazenar e buscar embeddings (ex: Pinecone, Qdrant).
